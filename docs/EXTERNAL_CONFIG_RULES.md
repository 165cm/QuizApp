## 📝 外部設定のドキュメント化ルール

**重要**: コードで直接管理できない外部サービスの設定は、必ず `docs/` フォルダにドキュメントとして残すこと。

### 対象となる外部設定

1. **Supabase**
   - テーブル構成（カラム、型、制約）
   - RLSポリシー
   - 認証設定
   - → `docs/SUPABASE_SPEC.md` に記載

2. **GAS (Google Apps Script)**
   - プロキシURL
   - スクリプトプロパティ（APIキー等）
   - レート制限設定
   - → `docs/GAS_SPEC.md` に記載

3. **その他の外部サービス**
   - Firebase, Vercel, AWS等を使用する場合
   - 環境変数の一覧
   - 認証設定

### ドキュメント更新タイミング

- 外部設定を変更した時は**必ず同時に**ドキュメントを更新
- コミットメッセージに「📝 外部設定更新」を含める
